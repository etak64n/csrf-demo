何が起きたのか（CSRF 攻撃の解説）

  1. あなたは SecureBank (localhost:3000) にログイン済みで、セッション Cookie を持っていました。
  2. この攻撃者サイト (localhost:8080) にアクセスしました。
  3. 「今すぐ受け取る」をクリックすると、隠しフォームが SecureBank の送金 API (/api/transfer) に POST リクエストを送信しました。
  4. ブラウザは localhost 宛のリクエストにセッション Cookie を自動的に付与するため、SecureBank はあなた本人からのリクエストだと判断しました。
  5. 結果、あなたの口座から attacker に ¥50,000 が送金されました。

対策: CSRF トークン、SameSite Cookie 属性 (Strict)、Origin ヘッダーの検証などで防げます。
